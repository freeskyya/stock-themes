Theme 01

70 人公司導入 Agent 後，CPU / Server 預算報告

假設一間 70 人的 SoC design house，平均每人約 3 個 agent workflow，估算因此新增的企業基礎設施需求。

Internal Budget View
Executive Summary

核心摘要

對一間約 70 人的 SoC design house 來說，若平均每人導入約 3 個 agent workflow，企業通常不會先大規模購買 GPU 伺服器， 而是先補 CPU-heavy 的 infrastructure，用來支撐 orchestration、知識檢索、文件索引、log 分析、build / test 與內部流程自動化。 在雲端為主情境下，合理新增需求約為 1–3 台像樣的 server / workstation；若走 hybrid / 部分本地 AI，則會進一步增加 CPU、GPU 與 storage 投入。

公司規模

70 人

平均 agent / 人

3 個

總 agent workflow

210 個

同時活躍 agent

約 20–40 個

Key Assumptions

分析假設

  • agent 主要協助研發、文件處理、內部檢索、bug / issue summary 與流程自動化。
  • 並非所有 agent 同時滿載，估算同時活躍率約 10%–20%。
  • 70% 活躍 agent 屬於輕量 orchestration / retrieval 類工作，20% 屬於中度分析，10% 才是較重任務。
  • 公司偏向先補 CPU / RAM / NVMe，再視資料敏感度與頻率決定是否加本地 AI 節點。
Key Takeaways

關鍵判斷

  • agent 化會先推升 CPU-heavy infrastructure 需求，而不是一開始就全面推爆 GPU 伺服器。
  • 雲端為主情境勝在低 CAPEX 與導入速度；Hybrid 情境勝在資料掌控與長期能力建設。
  • 對投資敘事來說，受惠的不只 GPU，還包括 server CPU、memory、SSD 與 enterprise infrastructure。

情境 A：大多數模型走雲端 API

目的： 以最低本地 infrastructure 成本，快速導入 agent workflow，先提升員工生產力與流程自動化程度。

方法： 把重模型推理交給外部雲端 API，本地只保留 agent orchestration、文件檢索、權限控管、workflow 排程與內部工具整合。

項目估算
Agent orchestration / docs / search 主機1–2 台
每台規格16–32 cores / 64–128GB RAM / 1–4TB NVMe
預算級別約 US$8k–20k
  • 上線快、維運簡單。
  • 適合先驗證 agent 導入 ROI。
  • 缺點是對外部 API 依賴高，敏感資料治理要更小心。

情境 B：Hybrid / 部分本地 AI

目的： 在維持 agent 生產力的同時，提高對敏感資料、內部知識庫與高頻工作流的掌控度，並為長期擴充建立內部 AI infrastructure。

方法： 除了保留 workflow / search / orchestration 本地能力外，額外部署 CPU-heavy nodes 與小型 GPU 節點，支援私有文件檢索、embedding、reranking、部分模型 inference 與低延遲 agent 任務。

項目估算
CPU-heavy 機器2–4 台
CPU 規格24–64 cores / 128–256GB RAM / 2–4TB NVMe
GPU 節點1 台 shared AI node，1–2 張 inference-oriented GPU
Storage4TB–12TB 熱資料 NVMe，外加冷資料 / 備份層
預算級別約 US$20k–50k+
  • 較適合內部資料敏感、使用頻率高、希望降低長期外部依賴的公司。
  • 優點是掌控度更高、延遲更低、可客製化程度更好。
  • 缺點是前期 CAPEX 與維運複雜度都更高。

粗略預算拆分（中度導入版本）

CPU / Workstation / Server US$10k–22k
RAM / NVMe / Storage US$4k–10k
Search / DB / Orchestration infra US$3k–8k
Optional small GPU node US$6k–15k
Ops / backup / misc US$2k–6k

Investment Takeaway

對投資敘事的意義

  • 這類需求不是靠單一超大單，而是靠大量中小企業同步補 infrastructure。
  • 最先受惠的不是只有 GPU，還有 server CPU、memory、SSD、networking。
  • 對 AMD 來說，若企業 agent 化普及，EPYC / Ryzen Pro / workstation 類產品都可能受益。
Recommendation

一句結論

對 70 人公司來說，agent 化不是科幻，而是足以合理新增 1–3 台 CPU-heavy 機器的 IT 預算項目； 單家公司不誇張，但大量公司一起做，對 CPU 市場就很有感。
