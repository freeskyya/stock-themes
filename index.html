Subtheme 01

100 人公司導入 Agent 後，CPU / Server 預算報告

假設一間 100 人的 SoC design house，平均每人約 3 個 agent workflow，估算因此新增的企業基礎設施需求。

最後更新：2026-04-04 18:57 CST

Internal Budget View
Executive Summary

核心摘要

對一間約 100 人的 SoC design house 來說，若平均每人導入約 3 個 agent workflow，企業通常不會先大規模購買 GPU 伺服器， 而是先補 CPU-heavy 的 infrastructure，用來支撐 orchestration、知識檢索、文件索引、log 分析、build / test 與內部流程自動化。 在雲端為主情境下，合理新增需求約為 2–4 台像樣的 server / workstation；若走 hybrid / 部分本地 AI，則會進一步增加 CPU、GPU 與 storage 投入。

公司規模

100 人

平均 agent / 人

3 個

總 agent workflow

300 個

同時活躍 agent

約 30–60 個

Key Assumptions

分析假設

  • agent 主要協助研發、文件處理、內部檢索、bug / issue summary 與流程自動化。
  • 並非所有 agent 同時滿載，估算同時活躍率約 10%–20%。
  • 70% 活躍 agent 屬於輕量 orchestration / retrieval 類工作，20% 屬於中度分析，10% 才是較重任務。
  • 公司偏向先補 CPU / RAM / NVMe，再視資料敏感度與頻率決定是否加本地 AI 節點。
Key Takeaways

關鍵判斷

  • agent 化會先推升 CPU-heavy infrastructure 需求，而不是一開始就全面推爆 GPU 伺服器。
  • 雲端為主情境勝在低 CAPEX 與導入速度；Hybrid 情境勝在資料掌控與長期能力建設。
  • 對投資敘事來說，受惠的不只 GPU，還包括 server CPU、memory、SSD 與 enterprise infrastructure。

情境 A：大多數模型走雲端 API

目的： 以最低本地 infrastructure 成本，快速導入 agent workflow，先提升員工生產力與流程自動化程度。

方法： 把重模型推理交給外部雲端 API，本地只保留 agent orchestration、文件檢索、權限控管、workflow 排程與內部工具整合。

項目估算
Agent orchestration / docs / search 主機數量2–3 台（整體配置）
每台 CPU 規格16–32 cores / 64–128GB RAM / 1–4TB NVMe（每台）
每台節點預算約 US$4k–9k / 台
預算級別約 US$12k–28k（整體預算）
  • 上線快、維運簡單。
  • 適合先驗證 agent 導入 ROI。
  • 缺點是對外部 API 依賴高，敏感資料治理要更小心。

情境 B：Hybrid / 部分本地 AI

目的： 在維持 agent 生產力的同時，提高對敏感資料、內部知識庫與高頻工作流的掌控度，並為長期擴充建立內部 AI infrastructure。

方法： 除了保留 workflow / search / orchestration 本地能力外，額外部署 CPU-heavy nodes 與小型 GPU 節點，支援私有文件檢索、embedding、reranking、部分模型 inference 與低延遲 agent 任務。

項目估算
CPU-heavy 機器數量3–5 台（整體配置）
每台 CPU 規格24–64 cores / 128–256GB RAM / 2–4TB NVMe（每台）
GPU 節點1 台 shared AI node，1–2 張 inference-oriented GPU（整體配置）
Storage8TB–20TB 熱資料 NVMe，外加冷資料 / 備份層（整體配置）
每台 CPU-heavy node 預算約 US$6k–12k / 台
GPU node 預算約 US$8k–20k / 台
Storage / backup 預算約 US$4k–10k（整體預算）
預算級別約 US$30k–70k+（整體預算）
  • 較適合內部資料敏感、使用頻率高、希望降低長期外部依賴的公司。
  • 優點是掌控度更高、延遲更低、可客製化程度更好。
  • 缺點是前期 CAPEX 與維運複雜度都更高。

Investment Takeaway

對投資敘事的意義

  • 這類需求不是靠單一超大單，而是靠大量中小企業同步補 infrastructure。
  • 最先受惠的不是只有 GPU，還有 server CPU、memory、SSD、networking。
  • 對 AMD 來說，若企業 agent 化普及，EPYC / Ryzen Pro / workstation 類產品都可能受益。
Recommendation

一句結論

對 100 人公司來說，agent 化不是科幻，而是足以合理新增 2–4 台 CPU-heavy 機器的 IT 預算項目； 單家公司不誇張，但大量公司一起做，對 CPU 市場就很有感。

Executive Summary

核心摘要

若未來幾年全球有約 50 萬～150 萬家知識密集型公司逐步導入 agent workflow，則企業新增的 CPU-heavy infrastructure 需求可能達到 100 萬～300 萬台等價設備， GPU / AI node 可能達 10 萬～50 萬台，對應的總體 infrastructure 支出可能落在約 200～500 億美元級別。這條敘事若成立，受惠的不只 GPU，也包含 server CPU、memory、SSD 與 enterprise infrastructure。

Model Assumptions

估算假設

  • 不是全球所有公司都會 agent 化，主要 relevant universe 為知識密集、可流程自動化的企業。
  • 小型公司以雲端為主，但仍可能補 1–3 台 CPU-heavy machine；中大型公司則逐步增加 hybrid / on-prem 比重。
  • 新增 infrastructure 不只支撐模型推理，也支撐 orchestration、retrieval、indexing、log analysis、workflow engine、storage 與內部控制層。
Key Interpretation

如何解讀

  • 這不是某家 hyperscaler 的單一大單，而是大量企業同步補 infrastructure 的 aggregate effect。
  • 短期 GPU 仍是 AI 主敘事，但 agent 化的企業內部 infrastructure 會讓 CPU / memory / storage 有更結構性的受惠邏輯。
  • 真正要看的是 adoption breadth，而不是單一 agent 功能本身有多炫。

Investment Takeaway

投資意義

  • 若 agent adoption 成為企業標配，受惠者將不只局限在 GPU，server CPU、memory、SSD、networking 與 enterprise platform 也會受益。
  • 對 AMD 來說，這條邏輯可同時支持 EPYC 與 workstation / enterprise CPU 的長期需求想像。
  • 真正值得追的不是單一新聞，而是：企業 agent adoption 的廣度、是否進入 hybrid 部署、以及是否形成持續性 infrastructure refresh cycle。

Subtheme 03

Agent 化趨勢下，AMD / Intel / Memory 受惠鏈比較

把企業 agent infrastructure 的受惠鏈拆成 CPU、GPU、memory、storage 與 enterprise infra，看看誰受惠最直接、誰比較像第二層受惠者。

最後更新：2026-04-04 18:57 CST

Beneficiary Map
Executive Summary

核心摘要

如果企業 agent adoption 成為中長期趨勢，最直接受惠者不只會是 GPU，還包括 server CPU、memory、SSD / storage 與 enterprise infrastructure。對 AMD 而言，受惠點較分散，可同時連接 EPYC、Ryzen Pro / workstation、以及 AI 平台敘事；Intel 則更偏向企業 CPU / enterprise installed base；memory 供應鏈則會隨 server / edge / storage 一起被拉動。

類別受惠程度原因

AMD

AMD 受惠點

  • EPYC：企業 agent infrastructure 若擴張，server CPU 需求是最直接受惠點。
  • Ryzen Pro / workstation：中型公司若用 workstation / edge node 撐 agent workflow，AMD 也有戲。
  • AI 平台敘事：若 hybrid 部署增加，AMD 還能把 CPU + AI / GPU + platform 故事一起講。

特點：不是只吃單點，而是 CPU + 平台敘事一起受益。

Intel

Intel 受惠點

  • 企業 installed base 很深，agent adoption 若發生在既有企業 IT 內，Intel 會先吃到一部分 refresh。
  • CPU-only / conservative deployment 的公司，常更容易先沿著既有 Intel 環境擴。
  • 如果市場主敘事是企業 CPU refresh，而不是 pure AI GPU，Intel 也不會缺席。

特點：比較像既有 enterprise CPU 基盤的受惠者。

Memory / Storage

Memory / Storage 受惠點

  • 每新增一台 CPU-heavy node，通常都會拉高 DRAM / NVMe 配置。
  • agent workflow 很吃文件索引、log、cache、vector store，storage 是很實在的需求。
  • 這條線不靠單一贏家，屬於「系統性一起受惠」。

特點：屬於第二層但很穩的受惠鏈。

Bottom Line

一句結論

如果 agent 化變成企業標配，AMD 的優勢在於可以同時吃到 server CPU 與平台敘事；Intel 受惠邏輯則更偏向企業 CPU refresh；而 memory / storage 供應鏈則會因整體機器規格上升而得到更廣、更穩的第二層受惠。
